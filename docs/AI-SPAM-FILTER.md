# OpenRouter AI 垃圾评论检测指南

## 概述

该功能集成 OpenRouter API 和 Claude 3.5 Sonnet 模型，自动检测和过滤博客评论中的垃圾内容、骚扰、广告等。

## 功能特性

- 🤖 **AI 智能检测** — 使用 Claude 分析评论内容、发起者信息
- ⚡ **两阶段检测** — 本地快速检查 + AI 深度分析，节省 API 调用次数
- 📊 **风险评分** — 0-100 分，自动标记为"安全"、"警告"、"危险"
- 🔍 **详细分析** — 记录每条评论的风险原因，便于审核
- 🎛️ **灵活配置** — 可随时启用/禁用，不影响评论正常使用
- 🔐 **自适应处理** — 高风险访客评论自动隐藏（已登录用户不受限制）

## 快速开始

### 1. 配置 OpenRouter API Key

在 `.env` 文件中添加：

```env
OPENROUTER_API_KEY=your_openrouter_api_key_here
NEXT_PUBLIC_SITE_URL=https://yourdomain.com
```

如何获取 API Key：
1. 访问 [OpenRouter](https://openrouter.ai/)
2. 注册账户
3. 在 API Keys 页面生成新的密钥
4. 复制并保存到 `.env`

### 2. 启用 AI 检测

进入 **后台 → 站点设置 → 评论设置**，勾选"启用 AI 垃圾评论检测"

### 3. 查看检测结果

进入 **后台 → 评论管理**，每条评论右侧会显示：
- 🤖 + 红/橙/绿色数字 — 表示风险评分
- 将鼠标悬停在评分标签上可看到风险原因

## 检测原理

### 阶段 1：本地快速检查（成本低，快速）

在调用 AI 前进行本地规则检查，包括：

| 检查项 | 分数 | 触发条件 |
|--------|------|---------|
| 链接数量 | 10-20 | 1-2 个链接；3+ 个链接 +20 分 |
| 内容长度 | 15 | 超过 5000 字 |
| 重复字符 | 10x次数 | 连续重复 5+ 个字符 |
| 垃圾关键词 | 30 | viagra, casino, 博彩等 |
| 邮箱风险 | 5 | 包含 + 号（如 user+spam@email.com） |
| 全大写内容 | 15 | 超过 50% 大写字母 |

**本地风险 ≥ 20 分** → 进入 AI 深度分析

### 阶段 2：AI 深度分析（更准确）

Claude 3.5 从 5 个维度评估评论：

1. **垃圾广告指数** — 商业链接、产品推销
2. **骚扰/人身攻击** — 辱骂、威胁内容
3. **链接重复/机器生成** — 大量外链、自动生成迹象
4. **与主题关联度** — 是否与文章相关
5. **语言质量** — 拼写错误、无意义文字

返回综合风险分数（0-100）

### 风险分数解释

| 分数范围 | 级别 | 颜色 | 处理方式 |
|---------|------|-----|---------|
| 0-19 | 安全 ✅ | 绿色 | 自动通过（需要审核时显示待审） |
| 20-59 | 警告 ⚠️ | 橙色 | 需人工审核 |
| 60-100 | 危险 🚫 | 红色 | 访客评论自动隐藏；登录用户仍需审核 |

## 配置选项

### 在站点设置中

```
评论设置 → 🤖 启用 AI 垃圾评论检测
```

**启用后**：
- 所有新评论提交时自动进行检测
- 在后台评论管理页面显示风险评分
- 高风险访客评论自动设为未显示状态

**禁用后**：
- 不再进行 AI 检测
- 现有的风险评分仍保留在数据库中

### 其他相关设置

- **评论需审核后显示** — 启用时，所有评论默认待审，AI 评分只作参考
- **前台显示评论 IP** — 开启后可在前台显示评论者 IP

## API 成本估算

OpenRouter 的 Claude 3.5 Sonnet 价格（按 1M tokens 计算）：

- 输入：$3/1M tokens
- 输出：$15/1M tokens

**单条评论成本**（估算）：
- 平均输入：~500 tokens（提示词 + 评论内容）
- 平均输出：~200 tokens（JSON 结果）
- **单条成本**：约 $0.003（0.3 美分）

**月度估算**（假设 1000 条评论）：
- 成本：约 $3（30 元人民币）
- 日成本：~0.1 美分

💡 **提示**：可通过以下方式降低成本：
1. 关闭 AI 检测，仅使用本地检查
2. 手动审查高风险评论后删除（避免 AI 再次检测）
3. 调整本地检查阈值（在代码中修改）

## 故障排除

### 提示缺少 OPENROUTER_API_KEY

检查 `.env` 文件中是否有效设置了密钥。务必：
- 从 OpenRouter 复制完整的密钥
- 不要包含引号
- 重启开发服务器或部署后重新加载

### AI 检测失败但评论仍然提交

这是设计行为。如果 AI 调用失败，系统会：
1. 回退到本地检查结果
2. 如果本地风险分数 > 0，记录该分数
3. 评论正常提交（不阻断用户）

查看日志：`[ai-detection] 分析失败: ...`

### 评论一直显示"待审"

原因可能是：
- **评论审核开启** — 所有评论默认待审（正常）
- **AI 检测标记为高风险** — 检查评论管理页面的风险分数

### OpenRouter API 超限

提示 `Too many requests` 或 `Rate limit exceeded`：
- OpenRouter 有速率限制
- 可升级账户获得更高配额
- 或临时禁用 AI 检测

## 自定义检测规则

要修改本地检查规则，编辑 `src/lib/openrouter-spam-filter.ts` 中的 `quickSpamCheck()` 函数：

```typescript
export function quickSpamCheck(content: string, authorEmail?: string) {
  let localRiskScore = 0

  // 示例：修改链接数量规则
  const linkCount = (content.match(/https?:\/\//g) || []).length
  if (linkCount >= 5) {
    localRiskScore += 30  // 改成 +30 分
  } else if (linkCount >= 2) {
    localRiskScore += 15  // 改成 +15 分
  }

  // ... 其他检查
  return { shouldAnalyze: localRiskScore >= 20, localRiskScore }
}
```

## 禁用功能

### 完全禁用 AI 检测

1. 在 `.env` 中注释掉 `OPENROUTER_API_KEY`
2. 在站点设置中禁用"启用 AI 垃圾评论检测"
3. 评论提交时会自动跳过 AI 检测

### 只保留本地检查

在 `src/app/api/comments/route.ts` 中，注释掉 AI 调用部分：

```typescript

```

## 常见问题

**Q: 为什么某个正常评论被标记为高风险？**

A: 可能原因：
- 包含多个链接
- 包含特定关键词（如"免费"、"赚钱"）
- 全大写内容
- 评论长度异常

可在后台手动审核并通过，AI 会学习改进。

**Q: 登录用户的评论会被 AI 检测吗？**

A: 会，但不会自动隐藏。登录用户的评论风险分数仅用于参考，不会因为 AI 检测而自动设为待审。

**Q: 能否针对特定用户禁用 AI 检测？**

A: 目前不支持。可通过修改代码实现，或在后台手动修改评论的 `riskScore` 字段。

**Q: 检测会影响评论提交速度吗？**

A: AI 调用通常在 1-2 秒内完成。如果感觉慢，可：
1. 禁用 AI 检测，只用本地检查
2. 在另一个线程/队列中异步执行检测
